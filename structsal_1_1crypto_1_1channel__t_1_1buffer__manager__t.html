Detailed Description

Callbacks for handshake/encrypted/decrypted buffers.

Channel allocates new buffers with alloc() and returns those buffers to transport layer with ready().

Callback to allocate new buffer.

On return, set buffer to pointer of new memory area with buffer_size where output message will be stored. If generated message does not fit into buffer, this callback is called multiple times for new chunks until whole message is stored. For each buffer, ready() will be called separately.

Return value uintptr_t can be used by application layer to tie each returned buffer with application-specific data. It is not used by channel except storing and returning back to application in call to ready().

Callback for stored message (or chunks if message is split between multiple buffers).

Buffer contains pointer returned by alloc() and buffer_size has number of actually stored bytes. It's value might be zero if channel did not generate any output, in which case buffer can be simply released.

For meaning of user_data, see alloc()
