Detailed Description

Asynchronous networking I/O completion service.

This class holds platform-dependent completion handler (IOCP/epoll/kqueue) but it is not meant to be used directly for polling completions. Instead, per-thread async_service_t::context_t does actual completion waiting and resources management.

async_service_t::context_t sal::net::async_service_t::make_context ( size_t  max_events_per_poll = 16)
inline

Create new I/O completion thread context.

max_events_per_poll configures how many events are batched when waiting for completions. If this number is too little, then underlying syscall must be called more often (with kernel- and user-mode switch overhead). On too big value and/or if each completion handling takes too long, it might take too long time to get to handling specific completion.
